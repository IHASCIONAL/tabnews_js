Os personagens

Quatro servidores,
um único objetivo

A resolução DNS é delegada em uma hierarquia. Cada ator sabe apenas o suficiente para apontar para o próximo.

① Resolver

Recursive Resolver

Seu ponto de partida — servidor do provedor ou público (ex: 8.8.8.8). Faz todas as perguntas por você.

② Root

Root Server (".")

O topo da hierarquia. Não conhece IPs finais, mas sabe quem cuida de cada extensão (.com, .br…).

③ TLD

TLD Server (Top-Level Domain)

Cuida de uma extensão (.com, .br). Sabe qual é o Authoritative Server do domínio solicitado.

④ Auth

Authoritative Server

O servidor do dono do domínio. Quem controla o domínio define qual servidor é o autoritativo — e é ele que guarda os registros DNS reais e retorna o IP final.

Esquema visual

Quem fala com quem?

O PC só conversa com o Resolver — que percorre toda a hierarquia de forma transparente.

O fluxo

Resolvendo tabnews.com.br

1

Você digita o endereço

O navegador pergunta ao Recursive Resolver: "Qual o IP de tabnews.com.br?"

2

Resolver → Root Server (".")

O Root não sabe o IP, mas responde: "O TLD .br é responsabilidade deste servidor aqui."

3

Resolver → TLD Server (.br)

O TLD não sabe o IP final, mas responde: "Quem tem os registros de tabnews.com.br é este Authoritative Server."

4

Resolver → Authoritative Server (resposta!)

O servidor do dono do domínio — configurado por quem registrou o site. Ele possui todos os registros DNS reais e retorna o IP final ao Resolver, que entrega ao seu navegador.

5

Navegador → Servidor final

Agora com o IP em mãos, o navegador faz a requisição HTTP diretamente ao servidor do TabNews.

Registro de domínios

Como um domínio entra no ar?

Antes de qualquer resolução acontecer, alguém precisou registrar o domínio — um fluxo diferente, de escrita.

1

Registrante → Registrar (ex: registro.br)

Você solicita a reserva de meusite.com.br. O Registrar é a empresa autorizada a receber pedidos de registro.

2

Registrar → Registry (NIC.br)

O Registrar repassa os dados para o Registry — entidade que administra a zona .br. Para domínios .br, o Registry é operado pelo NIC.br.

3

Registry atualiza a zona TLD (.br)

O NIC.br insere um registro NS na zona .br. O valor padrão aponta para um servidor autoritativo do próprio registro.br — ex: a.auto.dns.br.

4

Domínio ativo na zona

Qualquer consulta DNS para meusite.com.br agora encontra o NS record na zona .br e chega ao Authoritative Server. Você pode trocar o NS para um servidor próprio a qualquer momento.

Nota: Para domínios .br, o registro.br atua como Registrar e o NIC.br (mesma organização, CGI.br) é o Registry — uma particularidade do modelo brasileiro. Em domínios .com, Registrar (ex: GoDaddy) e Registry (Verisign) são entidades completamente distintas.

Performance

Por que não é lento?
Cache + TTL

Fazer esse caminho inteiro a cada acesso seria impraticável. Por isso, os resultados são armazenados em cache.

🖥️ Seu computador

Guarda os IPs de sites visitados recentemente na memória local.

📡 Seu roteador

Mantém um cache compartilhado para todos os dispositivos da rede.

🌐 Recursive Resolver

Armazena respostas para as milhares de consultas que recebe por segundo.

TTL — Time To Live

Cada registro DNS vem com um "prazo de validade". Enquanto o TTL não expirar, o cache é usado. Quando expira, a busca recomeça.

Mão na massa

Consultando o DNS na prática

Com o pacote dnsutils você consegue fazer requisições DNS direto do terminal e ver os registros do Authoritative Server.

# instalar no Ubuntu/Debian
sudo apt install dnsutils
# endereço IP do domínio
dig ihascional.com.br A

;; ANSWER SECTION:
ihascional.com.br. 300 IN A 104.21.x.x
# registros de texto (SPF, verificações…)
dig ihascional.com.br TXT

;; ANSWER SECTION:
ihascional.com.br. 300 IN TXT "v=spf1 …"

O tipo do registro (A, TXT, MX, CNAME…) é exatamente o que você configura no painel do Authoritative Server — o servidor do dono do domínio.

Dica de ouro

O ponto final invisível

Todo domínio termina com um ponto — o Root Server. Os navegadores modernos o ocultam, mas ele é o ponto de partida técnico de toda a árvore DNS.

hostwww . domíniotabnews . TLDcom.br root (oculto).

A resolução acontece da direita para a esquerda: primeiro o Root (.), depois o TLD (.com.br), depois o domínio (tabnews), e por fim o host (www).
